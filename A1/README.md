## Описание файлов

- **`a1.cpp`**  
  Реализует два варианта метода Монте-Карло:
  - *Способ 1*: Генерация точек в широкой области, охватывающей все три круга
  - *Способ 2*: Генерация точек в узкой области, охватывающей только пересечение
  Сохраняет результаты в `esults.csv` с шагом по `N` от 100 до 100 000

- **`results.csv`**  
  Содержит экспериментальные данные:
  - `N` — количество сгенерированных точек
  - `res1` — оценка площади для способа 1
  - `res2` — оценка площади для способа 2

- **`draw_plots.py`**  
  Анализирует данные:
  1. Вычисляет точное значение площади 
  2. Генерирует два графика:
     - `plot1.png` — динамика оценок площади в зависимости от `N`,
     - `plot2.png` — относительная ошибка (%) в линейном масштабе.



## Вывод
Чем больше точек- тем точнее результат. Также немаловажную роль играет рассматриваемая область генерации точек, чем она точнее (рамка описана вокруг пересечения кругов), тем быстрее сойдётся результат, поскольку вероятность попадения точки в пересечение кругов выше. В целом, уже при `N=55000` способ 2 выдаёт стабильный результат с точностью до сотых, в то время как способу 1 для достижения такого же результата нужно гораздо больше точек.
